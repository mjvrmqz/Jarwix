# CONTACTS — How to Use the Contacts Database

## Overview
MJ's contacts now live in the **Contacts database** on the Dashboard page in Notion (ID: 36420c51-aebe-80ca-bdbe-d632050b2e34). Claude queries this database whenever a person is involved in scheduling. This file explains how to use that data and where Claude logs observations.

## How to Use the Contacts Database
Each row has a Name, Relationship, and Notes. Use this to understand who each person is and their typical behavior patterns before acting on any iMessage or scheduling anything that involves them.

Use Cases:
- When checking iMessages and seeing a message from a contact, cross-reference here to understand if the message is routine or unusual based on their patterns.
- When suggesting a task that involves a contact (gym with Jairo, chores Azucena mentioned), check iMessages first and then use Notes here to judge urgency and context.
- When a contact reaches out about something, consider whether it should affect the schedule before building it.

## Where Claude Logs Contact Observations
Claude logs ongoing observations about contacts directly in the Notes field of each contact's row in the Contacts database. Keep notes compact and informative. Add anything relevant — behavioral patterns, typical timing, recurring themes in messages.

Claude also logs contact-related events in HISTORY.md when they affect the schedule.

## Associates vs Acquaintances
Associates — personal contacts (family, friends). Acquaintances — work contacts (clients, collaborators). Both live in the same Contacts database, distinguished by the Relationship field.
