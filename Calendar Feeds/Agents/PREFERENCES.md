# PREFERENCES — How to Use the Preferences Database

## Overview
MJ's scheduling preferences now live in the **Preferences database** on the Dashboard page in Notion (ID: 36420c51-aebe-8094-9bab-ddbe11d22c94). Claude reads that database at the start of every session. This file explains how to interpret and act on that data.

## How to Read the Preferences Database
Each row has a Question (what the preference is about), an Answer (MJ's actual preference), and a Type (Work Hours / Meetings & Calls / Task Batching). Read all rows and build a mental model of MJ's scheduling preferences before making any scheduling decisions.

## Night Owl Note — Critical for Auto Scheduler
MJ's peak focus and creative hours are 5PM–4AM. His morning hours suit admin and low-demand work. This directly affects window logic:
- **Morning Peak** → admin, calls, organization, review only
- **Midday Dip** → maintenance, admin, organization
- **Afternoon Rebound** → MJ's real peak. Creative, flow, analytical, research, learning tasks go here.

Do NOT put deep work in Morning Peak regardless of what the window name implies.

## Soft Rules
Preferences are soft — Claude respects them by default but can override when MJ explicitly asks or when a hard constraint demands it. If a preference conflicts with good judgment given MJ's actual day, use good judgment and explain why.
